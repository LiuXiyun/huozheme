# 活着么 H5 MVP PRD

## 1. 产品定位

活着么是一个面向年轻用户的情绪测试 H5。用户用 10-20 秒完成今日状态测试，获得一张可截图、可保存、可转发的精神存活报告。

核心目标不是心理评估，而是制造可传播的身份表达：让用户觉得“这说的就是我”，并愿意把结果发给朋友。

## 2. 用户与场景

目标用户：
- 打工人：上班压力、通勤、会议、余额焦虑。
- 大学生：熬夜、ddl、社交、考试。
- 独居人：作息、孤独、生活自理、深夜状态。
- 自由职业者/创业者：收入波动、自我驱动、客户压力。

核心使用场景：
- 早上/午休/深夜刷到链接，随手测一次。
- 在微信群、朋友圈、小红书看到朋友的结果图后进入。
- 用户想用一张好笑的卡片表达今日精神状态。

## 3. 页面级需求

### 3.1 首页/设置页

目标：3 秒内让用户理解这是一个今日状态测试，并开始答题。

内容：
- 产品名：活着么。
- 今日氛围：展示“今日检测人数”“平均存活指数”“城市异常播报”等轻量数据。
- 用户设置：城市、身份主题、今日状态。
- 主题入口：打工人、大学生、独居人、自由职业者四类优先。
- 主按钮：开始今日检测。
- 结果预览：一张小型报告样张，强化“会生成可分享卡片”。

成功标准：
- 首屏不出现复杂说明。
- 移动端无横向滚动。
- 用户最多 3 次点击进入答题。

### 3.2 答题页

目标：让用户无压力完成 5 题，并通过文案持续产生共鸣。

内容：
- 题目进度：1/5 到 5/5。
- 当前主题标签、城市、题库来源。
- 单选答案，每题 4 个选项。
- 退出成本低，不设置返回确认。
- 题目来源：优先使用每日 AI 题库池；失败时使用本地大题库兜底。

交互：
- 点击答案后自动进入下一题。
- 末题后直接进入结果页。
- 选项文案要短、像弹幕，不像问卷。

成功标准：
- 单题操作不超过 2 秒。
- 题目和选项在常见手机视口内不互相遮挡。

### 3.3 结果页

目标：让用户立即想截图或保存。

内容：
- 今日精神存活指数。
- 状态标题。
- 毒舌吐槽。
- 城市/身份排名。
- 今日建议。
- 诊断指标：精神电量、离线风险、复活概率。
- 今日死因/复活方式/适合发圈文案。
- 分享图工厂：报告卡、朋友圈、小红书三种模板。
- 毒舌强度：轻吐槽、够真实、黑化版。
- 操作：换一句、保存当前海报、复制文案、再测一次。

传播机制：
- 结果包含身份和城市标签。
- 分享文案包含可复制句子。
- 报告卡竖版，适合朋友圈/小红书/群聊截图。

成功标准：
- 结果卡在移动端首屏露出核心信息。
- 保存图与页面卡片信息一致。

### 3.4 解锁弹层

目标：先验证付费兴趣，不接真实支付。

内容：
- 付费项：年度精神账单、黑化主题卡、朋友圈文案加强版。
- 价格锚点：9.9 元。
- 付费预览：年度崩溃热力图、人格化死因、朋友圈加强版数量。
- 当前状态：内测预约/占位。

成功标准：
- 用户点击后能理解这是一个未来付费点。
- 不假装已经完成支付。

### 3.5 分享图

目标：生成一张独立成立的传播卡。

内容：
- 产品名与日期。
- 分数、状态、吐槽、排名、建议。
- 今日死因、复活方式、可直接转发的那句。
- 模板风格：报告卡、朋友圈、小红书。
- 入口标识：活着么。

成功标准：
- 图片比例 3:4，适合社交平台。
- 文字不超出画布。

## 4. 数据与规则

题库和结果采用前端兜底 + 服务端增强：
- 本地每个主题保留大题库，用日期和主题稳定抽取 5 题。
- 部署后 `/api/questions` 每日为主题和城市生成 AI 题库池并缓存。
- 分数由答题权重、主题、日期盐值计算。
- 排名为算法生成的娱乐数据，不展示“真实统计”字样。
- 今日检测人数、平均分和活跃城市通过 `/api/stats` 展示“每日冷启动基数 + 真实增量”，真实事件通过 `/api/event` 记录。
- 本地存储仅保存最近测试记录，不收集真实姓名、手机号、公司、定位。

## 5. 商业化

MVP 阶段只做入口占位：
- 9.9 元隐藏结局。
- 6.6 元主题卡。
- 品牌广告位预留在结果页侧栏和分享图底部。
- 通过点击“预约 9.9 内测”和“记录我的内测名额”记录早期付费意向，并用预约人数作为转化信号。

付费转化设计：
- 免费结果给出完整但不过量的信息。
- 付费卡片先露出一个具体洞察，让用户知道解锁后会看到什么。
- 分享图工厂让用户在结果页多停留、多操作，增加付费入口曝光。

上线验证指标：
- 完测率：60%+。
- 保存/复制率：20%+。
- 分享入口点击率：15%+。
- 次日回访：10%+。
