# 活着么 UI 设计稿说明

## 1. 设计方向

关键词：深夜检测站、精神电量、黑色幽默、城市故障播报。

视觉不做治愈系，也不做医疗感。整体应该像一张“地下测试报告”：深色底、强对比文字、荧光强调色、网格/扫描线、紧凑信息块。

## 2. 色彩

主背景：`#101114`、`#17181d`  
主文字：`#f5f2e8`  
次文字：`#a7a8aa`  
强调色：
- 酸性绿 `#c7ff4f`：主按钮、分数、关键指标。
- 珊瑚红 `#ff6b57`：风险、状态、热度。
- 青色 `#54d6d1`：辅助信息、城市感。
- 黄色 `#ffd166`：高级/付费点。

避免大面积单色渐变。页面要有黑、绿、红、青、黄的层次。

## 3. 字体与排版

字体：系统无衬线，中文优先使用苹方/微软雅黑。  
标题：粗、短、冲击强。  
正文：短句化，避免长段解释。  
按钮：稳定高度 54-58px，圆角 8px。  
卡片：只用于结果报告、设置模块、弹层，不做层层嵌套。

## 4. 页面布局

### 首页

桌面端：
- 左侧为品牌标题、状态选择和 CTA。
- 右侧为报告预览与今日数据。
- 首屏必须看到主要 CTA。

移动端：
- 单列布局。
- 标题、CTA、设置项优先展示。
- 报告预览在首屏下方露出一部分。

### 答题页

- 居中单卡片。
- 顶部显示进度、主题、城市和题库来源。
- 选项是大点击区，适合拇指操作。
- 题目字号比首页小，避免压迫。

### 结果页

桌面端：
- 左侧为竖版报告卡。
- 右侧为分享文案、分享图工厂、操作区、解锁入口、复访提示。

移动端：
- 报告卡在上，操作区在下。
- 主按钮固定视觉优先级：保存图 > 复制文案 > 再测。
- 分享图工厂使用竖版预览，模板和语气按钮必须易点。

### 解锁弹层

- 居中覆盖。
- 不进入支付，只展示价值点和内测预约反馈。
- 可关闭，不阻断主流程。

## 5. 组件规范

按钮：
- 主按钮：酸性绿底、黑字。
- 次按钮：透明底、浅边框。
- 付费按钮：黄绿色强调，不抢主流程。

主题选择：
- 使用分段卡片/按钮，不用纯下拉。
- 选中态要明显，改变边框和背景。

报告卡：
- 固定竖版比例感。
- 分数为最大视觉焦点。
- 排名与建议使用信息块。
- 底部保留产品标识和分享口令。

分享图工厂：
- 使用 3:4 预览图。
- 模板按钮：报告卡、朋友圈、小红书。
- 语气按钮：轻吐槽、够真实、黑化版。
- 切换后立即刷新预览，制造可把玩感。

功能图：
- 首页每个核心功能都需要有小型图形化预览。
- 图形以 CSS/Canvas 生成，避免依赖外部图片资源。
- 图形必须说明“报告图、城市榜、隐藏结局”三种核心价值。

Toast：
- 底部浮层。
- 不超过 2 秒。

## 6. 内容语气

可以毒舌，但不恐吓，不做真实诊断，不碰医疗承诺。

好文案标准：
- 像朋友吐槽，不像 AI 总结。
- 能截图单独传播。
- 有身份标签和城市标签。
- 避免攻击具体群体。
